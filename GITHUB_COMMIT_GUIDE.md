# GitHub Commit Guide for Fourfold Fate

## Option 1: Using GitHub Desktop (Easiest)

1. **Download GitHub Desktop** (if not installed):
   - Go to: https://desktop.github.com/
   - Install it

2. **Open your project in GitHub Desktop**:
   - File → Add Local Repository
   - Browse to: `C:\Users\arset\Fourfold Fate`
   - Click "Add repository"

3. **If repository doesn't exist on GitHub yet**:
   - Click "Publish repository" button
   - Choose a name (e.g., "Fourfold-Fate")
   - Make it Private or Public
   - Click "Publish repository"

4. **Commit and Push**:
   - You'll see all your changed files
   - Write a commit message: "Initial setup: Core systems, UI, and scene setup"
   - Click "Commit to main"
   - Click "Push origin" to upload to GitHub

1. Download from: https://git-scm.com/download/win

3. Restart your terminal/command prompt

### Step 4: Add Files and Commit
```bash
# Navigate to project folder
cd "C:\Users\arset\Fourfold Fate"

# Initialize git repository (if not already done)
git init

# Add all files
git add .

# Commit with message
git commit -m "Initial setup: Core systems, UI, and scene setup"

# Add remote (replace 'Fourfold-Fate' with your desired repo name)
git remote add origin https://github.com/arset/Fourfold-Fate.git

# Push to GitHub
git branch -M main
git push -u origin main
```

**Note:** Make sure you've created the repository on GitHub first at https://github.com/new
- Repository name: `Fourfold-Fate` (or whatever you want)
- Make it Private or Public
- Don't initialize with README (since you already have files)

## Option 3: Using Unity's Built-in Version Control

1. **Enable Version Control in Unity**:
   - Edit → Project Settings → Version Control
   - Set "Version Control Mode" to "Visible Meta Files"
   - Set "Asset Serialization" to "Force Text"

2. **Use Unity Collaborate** (if available):
   - Window → General → Services
   - Enable Collaborate
   - Publish to Unity Cloud

## Recommended: GitHub Desktop

**GitHub Desktop is the easiest option** - it's visual, handles .gitignore automatically, and makes commits/pushes simple.

---

## Important Files to Commit

✅ **DO Commit:**
- All scripts in `Assets/Scripts/`
- Scene files in `Assets/Scenes/`
- Prefabs in `Assets/Prefabs/`
- Project settings (if using text serialization)

❌ **DON'T Commit:**
- `Library/` folder
- `Temp/` folder
- `Obj/` folder
- `Build/` folder
- `.csproj` files (Unity generates these)
- `.sln` files

The `.gitignore` file above will automatically exclude these.
